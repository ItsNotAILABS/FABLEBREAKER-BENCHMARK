Per-Family Scoring and Statistical Confidence in Multi-Family Benchmark Architectures

Freddy Medina
ItsNotAI LABS Research Division
June 2026

Abstract

Multi-family benchmarks evaluate candidates across structurally diverse expression classes, each targeting distinct optimization failure modes. Aggregate scoring—reporting a single speedup ratio across all families—can mask family-specific failures where a candidate excels on simple families while failing on adversarial ones. This paper presents a per-family scoring architecture for FableBreaker that decomposes certification results by adversarial family, reports family-conditional speedup ratios, and provides 95% confidence intervals for timing measurements. We formalize the statistical model underlying per-family scoring, derive minimum sample size requirements for reliable family-level inference, and show that per-family decomposition increases the diagnostic value of certification reports by enabling targeted identification of optimization weaknesses. The architecture is backward-compatible with existing aggregate scoring while providing strictly more information to both candidate authors and benchmark consumers.

Keywords: per-family scoring, confidence intervals, multi-family benchmarks, statistical measurement, diagnostic scoring, performance decomposition

1. Introduction

The FableBreaker benchmark evaluates candidates across multiple adversarial families, each designed to expose a distinct class of optimization failure. The existing scoring protocol reports aggregate metrics: total correctness count, overall speedup ratio, and timing percentiles across all cases. While sufficient for binary certification (pass or fail), aggregate scoring provides limited diagnostic information. A candidate that achieves 2.3× speedup overall might achieve 4× on modular_arithmetic_net (simple loop optimization) while producing incorrect results on 15% of overflow_corridor cases. The aggregate report obscures this critical distinction.

Per-family scoring addresses this limitation by decomposing all metrics by adversarial family. Each family receives its own correctness count, speedup ratio, timing distribution, and confidence interval. This enables candidate authors to identify exactly which family their optimization fails on, guides targeted improvement, and provides benchmark consumers with a detailed performance profile rather than a single number.

2. Statistical Model

2.1 Timing Distribution Assumptions

Within a single family, evaluation times are approximately log-normally distributed due to the multiplicative nature of nested computation costs. We model the timing for case i in family f as ti,f = exp(μf + σf · zi) where zi is a standard normal variate. This model justifies the use of geometric means and log-space confidence intervals for family-level timing reports.

2.2 Confidence Interval Construction

For each family, we compute a 95% confidence interval for the mean candidate evaluation time using the t-distribution approximation: CI = x̄ ± t0.025,n-1 · s/√n. For families with fewer than 30 cases in a single dataset, we note that the confidence interval may be wide and flag the measurement as preliminary. The minimum recommended sample size for reliable family-level inference is 20 cases per family, which FableBreaker achieves through its stratified generation process.

2.3 Speedup Significance

A family-level speedup is considered statistically significant if the confidence intervals of the candidate and baseline timing distributions do not overlap. This prevents reporting spurious speedups due to measurement noise on families with high timing variance.

3. Architecture

3.1 Scorer Modifications

The scorer maintains per-family accumulators during evaluation. After all cases are processed, the family_breakdown function computes per-family statistics including: case count, correct count, pass rate, speedup (only for families with 100% correctness), median timing, p95 timing, p99 timing, and 95% confidence interval. The breakdown is included in the scoring output as a nested object keyed by family name.

3.2 Report Schema Extension

The scoring report extends the existing schema with a family_breakdown field:

{
  "family_breakdown": {
    "dynamic_match_storm": {
      "count": 32,
      "correct": 32,
      "pass_rate": 1.0,
      "speedup": 1.87,
      "candidate_median_ms": 0.42,
      "candidate_p95_ms": 1.13,
      "candidate_p99_ms": 2.01,
      "ci_95_ms": [0.38, 0.51]
    },
    ...
  }
}

3.3 Backward Compatibility

Existing fields (aggregate speedup, overall correctness, failure list) remain unchanged. The family_breakdown is an additive extension that does not modify the interpretation of existing fields. Tools consuming the previous schema continue to function without modification.

4. Diagnostic Value

Per-family scoring enables several diagnostic workflows not possible with aggregate reporting:

  • Weakness identification: A candidate author can immediately see which family their optimization fails on and focus debugging effort accordingly
  • Optimization profiling: Different families stress different aspects of evaluation (iteration, recursion, structural navigation, arithmetic). Per-family speedups reveal which optimization strategies are effective
  • Regression detection: Between versions of a candidate, per-family comparison can detect regressions on specific families that would be masked by aggregate improvement
  • Benchmark evolution: When new families are added, per-family scoring immediately shows how existing candidates perform against new adversarial structures without reinterpreting aggregate scores

5. Conclusion

Per-family scoring with statistical confidence intervals transforms FableBreaker from a binary certification system into a diagnostic evaluation platform. Candidates receive actionable performance profiles; benchmark consumers receive fine-grained trust signals; and the benchmark itself gains the ability to evolve (adding families) without disrupting the interpretability of historical results. The architecture demonstrates that rigorous adversarial benchmarks benefit from the same statistical discipline applied in experimental science: report uncertainty, decompose by treatment, and enable targeted inference.
